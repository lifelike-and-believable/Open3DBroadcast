**Architecture Decisions:**
- Protocol: IETF draft-ietf-moq-transport-07 via moq-rs (Cloudflare)
- FFI Layer: Rust (moq-transport) ↔ C (FFI bindings) ↔ C++ (Unreal wrapper)
- Relay: moq-relay-ietf from moq-rs (production-ready, standards-compliant)
- Deployment: Relay-only (local for dev/test, CloudFlare for internet-scale)
- Transport Layer: WebTransport over QUIC (via quinn in moq-rs)
- Threading Model: Rust async/await bridged to Unreal game thread
- Track Strategy: Separate MoQ tracks for mocap data and audio

## Key Design Patterns

**moq-rs Integration:**
- FFI boundary: Manual C bindings + C++ wrappers
- Async bridging: Tokio runtime in dedicated thread
- Memory management: Careful ownership across FFI boundary
- Error handling: Rust errors translated to UE error codes

**Reference Implementations:**
- NNG Transport: Async threading, configuration parsing, UI patterns
- WebRTC Transport: Audio implementation, test structure
- moq-pub/moq-sub: Reference MoQ clients for interoperability testing

**UE 5.7 API Compliance:**
- All APIs verified against official documentation
- No "best guess" implementations allowed
- Threading on game thread for capture, FFI worker thread for moq-transport

**Standards Compliance:**
- IETF draft-ietf-moq-transport-07
- O3DS FlatBuffers schema (src/o3ds.fbs)
- O3DAudio encoding/decoding framework
- MoQ track naming conventions

## Benefits of moq-rs Integration

✅ **Standards Compliance:** Implements actual IETF MoQ specification
✅ **Interoperability:** Works with other MoQ clients/servers
✅ **Production Ready:** moq-rs is battle-tested by CloudFlare
✅ **CloudFlare Network:** Direct access to experimental relay network
✅ **Reduced Maintenance:** Protocol implementation maintained upstream
✅ **Future-Proof:** Tracks IETF standardization process

## Risk Management

**High-Priority Risks:**
- FFI layer complexity → Comprehensive tests, clear memory docs
- Rust toolchain setup → Pre-built binaries, CI/CD integration
- Async/await bridging → Proven patterns, dedicated tokio thread
- moq-relay-ietf deployment → Docker containers, clear deployment guide

**Contingency Plans:**
- FFI too complex → Subprocess with IPC fallback
- Rust build fails → Pre-built FFI binaries for Win64
- Relay problems → Local-only testing initially
- Standards compliance blocking → Best-effort compatibility
