# Product Management Reference Documentation

This directory contains detailed reference materials for the Product Manager Agent. These documents support the agent descriptor located at `.github/agents/product-manager-agent.md`.

## Documents

### [frameworks.md](frameworks.md)
Detailed product management frameworks for prioritization, research, and planning:
- **Prioritization**: RICE scoring, Value vs. Effort matrix, Kano model, MoSCoW method
- **Research**: JTBD, Value Proposition Canvas, Business Model Canvas, Porter's Five Forces, SWOT, TAM/SAM/SOM
- **Planning**: OKRs, North Star Metric
- **Pricing**: Pricing models and psychology

### [templates.md](templates.md)
Ready-to-use templates for product management activities:
- Market research report
- Feature proposal
- Market segment profile
- Positioning statement
- Competitive analysis
- Market opportunity assessment
- Feature handoff (for Planning Agent collaboration)

### [research-methodology.md](research-methodology.md)
Systematic approach to market research:
- 5-step research process
- Information sources (industry publications, technical communities, market research firms, academic sources, company sources)
- User research methods
- Competitive intelligence gathering
- Market trend research areas
- Common research pitfalls to avoid

### [entertainment-focus.md](entertainment-focus.md)
Specific focus on live real-time entertainment market:
- Target customer segments for Open3DStream
- Priority use cases (virtual concerts, character puppeteering, virtual production, interactive experiences)
- Entertainment market research focus areas
- Key research questions for entertainment industry

### [metrics-and-planning.md](metrics-and-planning.md)
Success metrics and planning cycles:
- Product metrics to track (adoption, engagement, quality, business)
- Market research effectiveness metrics
- Roadmap effectiveness metrics
- Quarterly planning cycle
- Feature development workflow (7-step process)
- Communication rhythm
- Decision framework

## Purpose

These reference documents were extracted from the main Product Manager Agent descriptor to:
1. **Reduce descriptor length** - The main descriptor is now ~240 lines (down from 1156 lines)
2. **Maintain detail** - All detailed methodologies, templates, and frameworks remain accessible
3. **Improve maintainability** - Reference materials can be updated independently
4. **Enable reuse** - Templates and frameworks can be referenced by other agents or users

## Usage

Product Manager Agents should reference these documents when:
- Conducting market research (see research-methodology.md)
- Prioritizing features (see frameworks.md)
- Creating deliverables (see templates.md)
- Planning quarterly cycles (see metrics-and-planning.md)
- Focusing on entertainment opportunities (see entertainment-focus.md)

The main agent descriptor (`.github/agents/product-manager-agent.md`) provides links to these documents in context.
