# Market Research Methodology

## Research Process

### Step 1: Define Research Questions
- What do we need to learn?
- What decisions will this inform?
- What are success metrics for the research?

### Step 2: Identify Information Sources

**Industry Publications:**
- VentureBeat, TechCrunch for tech industry news
- Variety, Hollywood Reporter for entertainment industry
- Gamasutra, GDC resources for game development
- VFX Voice, Animation Magazine for production

**Technical Communities:**
- Reddit (r/unrealengine, r/virtualproduction, r/gamedev)
- Discord servers for Unreal, Unity, motion capture
- Technical forums and discussion boards
- GitHub discussions and issue threads

**Market Research Firms:**
- Gartner, Forrester reports (when available)
- IDC market analyses
- Grand View Research industry reports

**Academic Sources:**
- ACM Digital Library
- IEEE Xplore
- SIGGRAPH proceedings

**Company Sources:**
- Competitor websites and documentation
- Product launch announcements
- User forums and support discussions
- Social media presence

### Step 3: Conduct Research
- Use web_search tool for current information
- Document sources and dates
- Cross-reference multiple sources
- Note conflicting information
- Track emerging patterns

### Step 4: Synthesize Findings
- Organize research by theme
- Identify key insights
- Highlight actionable findings
- Note confidence levels
- Document assumptions

### Step 5: Create Research Deliverables
- Executive summary
- Detailed findings report
- Competitive comparison matrices
- Market trend analysis
- Recommendations with rationale

## User Research Methods

### User Feedback Analysis
- Read reviews on G2, Capterra, Reddit
- Analyze GitHub issues and discussions
- Monitor social media sentiment
- Review forum discussions

### Workflow Documentation
- Study tutorial videos and documentation
- Identify common workflows
- Note pain points and workarounds
- Understand tool combinations

### Community Insights
- Track what users are building
- Identify popular use cases
- Note feature requests and wishlists
- Understand adoption barriers

## Competitive Intelligence

### Information to Collect

**Company Information:**
- Founding date and history
- Funding status and investors
- Company size and locations
- Leadership team background
- Strategic partnerships

**Product Details:**
- Feature comparison matrix
- Pricing and licensing models
- Technology stack (if known)
- Platform support
- Integration capabilities
- Performance characteristics

**Market Presence:**
- Target customer segments
- Case studies and customers
- Marketing positioning
- Social media engagement
- Community size
- Documentation quality

**Business Model:**
- Revenue streams
- Open-source vs. commercial mix
- Support offerings
- Ecosystem monetization
- Partnership models

## Market Trend Research

### Live Entertainment Technology
- Virtual concerts and performances
- Live streaming innovations
- Real-time audience interaction
- Hybrid live/virtual events
- Metaverse entertainment experiences

### Virtual Production
- LED wall technology (e.g., StageCraft)
- Real-time rendering advances
- Motion capture integration workflows
- Live broadcast integration
- Game engine adoption in production

### Motion Capture Technology
- Markerless capture systems
- Mobile/accessible mocap solutions
- AI-driven animation
- Real-time character animation
- Facial performance capture

### Streaming and Networking
- WebRTC innovations
- Low-latency protocols (WebTransport, QUIC, SRT)
- Cloud rendering and streaming
- 5G impact on live production
- Edge computing for real-time processing

## Common Research Pitfalls to Avoid

- ❌ Confirmation bias (only finding data that supports assumptions)
- ❌ Relying on single sources without corroboration
- ❌ Using outdated data (market moves fast, check dates)
- ❌ Assuming correlation implies causation
- ❌ Ignoring contrary evidence
- ❌ Over-generalizing from small samples
- ❌ Not documenting research methodology
