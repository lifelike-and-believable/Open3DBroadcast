# WebRTC Libraries for Open3DStream

## Quick Start: Build the Library Locally

**Important:** The pre-built library (`datachannel.lib` or `libdatachannel.a`) is **NOT included in the repository** due to its large size (~20-80 MB).

### To enable WebRTC support, run the build script:

**Windows:**
```bat
cd <repository-root>
build_webrtc_lib.bat
```

**Linux/macOS:**
```bash
cd <repository-root>
./build_webrtc_lib.sh
```

📖 **See [`BUILD_WEBRTC_LIBS.md`](../../../../BUILD_WEBRTC_LIBS.md) for detailed instructions, prerequisites, and troubleshooting.**

## What Gets Built

The build script will create:
- `datachannel.lib` (Windows) or `libdatachannel.a` (Linux/macOS) - Static library
- `datachanneld.lib` (Windows Debug, optional)
- `include/rtc/` - libdatachannel header files  
- `BUILD_INFO.txt` - Build metadata (version, date, configuration)

## How It Works

1. **Build Script** (`build_webrtc_lib.sh` or `.bat`) checks for OpenSSL
2. **Initializes** libdatachannel submodules (libjuice, usrsctp, etc.)
3. **Compiles** libdatachannel with OpenSSL crypto backend
4. **Copies** library and headers to this directory
5. **Unreal Plugin** automatically detects library at compile time
6. **WebRTC Features** enabled when library present

## Prerequisites

### All Platforms
- **CMake 3.13+** 
- **Git** (for submodules)
- **OpenSSL** (crypto library)

### Windows Specific
- **Visual Studio 2022** 
- **MSBuild** (included with VS)

### Installing OpenSSL

**Windows:**
- vcpkg: `vcpkg install openssl:x64-windows` (recommended)
- Chocolatey: `choco install openssl`
- Manual: https://slproweb.com/products/Win32OpenSSL.html

**Linux:**
- Ubuntu/Debian: `sudo apt-get install libssl-dev`
- Fedora/RHEL: `sudo dnf install openssl-devel`

**macOS:**
- Homebrew: `brew install openssl`

## Build Configuration

The library is built with these settings:
- **Static library** (no DLL dependencies)
- **NO_MEDIA=ON** (avoids libsrtp, reduces dependencies)
- **OpenSSL backend** (DTLS/SRTP encryption)
- **RelWithDebInfo** configuration (optimized with debug symbols)

## Conditional Compilation

The Unreal plugin uses conditional compilation:

```cpp
#ifdef O3DS_ENABLE_WEBRTC
  // WebRTC code enabled
#endif
```

This flag is automatically defined by `Open3DStream.Build.cs` when `datachannel.lib`/`libdatachannel.a` exists in this directory.

**Result:** Plugin compiles with or without WebRTC library, gracefully degrading if library is absent.

## Why Not Include Pre-built Libraries?

1. **Size** - 20-80 MB per platform would bloat the repository
2. **Platforms** - Would need Windows, Linux, macOS versions
3. **Updates** - Hard to keep in sync with libdatachannel updates
4. **Flexibility** - Developers can build with their preferred OpenSSL version

## Current Approach (Pre-built Libraries)

✅ **Works right now** - Build once, WebRTC enabled  
✅ **Simple** - Single command to build  
✅ **Flexible** - Use your own OpenSSL installation  
⚠️ **Manual step** - Developers must run build script locally

## Future Plan (Build from Source)

See **[Issue #10](https://github.com/lifelike-and-believable/Open3DStream/issues/10)** - Build mbedTLS from source

**Goal:** Build mbedTLS as a submodule so CI and developers can build WebRTC without external dependencies.

**Challenges:**
- mbedTLS 3.6.2 (stable) lacks DTLS-SRTP APIs
- Development branch has DTLS-SRTP but complex build requirements
- CMake integration with libdatachannel's `FindMbedTLS.cmake`

Once solved, this will enable fully self-contained builds.

## Troubleshooting

### "datachannel.lib not found"
→ Run `build_webrtc_lib.bat` or `./build_webrtc_lib.sh`

### "OpenSSL not found"
→ Install OpenSSL (see Prerequisites above)

### "WebRTC options don't appear in Unreal"
→ Check that library exists in this directory: `ls datachannel.lib` or `ls libdatachannel.a`

### "Build script fails"
→ See `BUILD_WEBRTC_LIBS.md` for detailed troubleshooting
